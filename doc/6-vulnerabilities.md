# Что такое уязвимость?

**Уязвимость** — любое несоответствие требованиям безопасности

Может быть:

- Случайной (ошибка разработчика)
- Намеренной (бэкдор, логическая бомба)

Курс фокусируется только на **уязвимостях в ПО**
(аппаратные тоже есть, но это другая история)

# Рост числа публичных уязвимостей

Современный мир зависит от ПО → количество известных уязвимостей растёт экспоненциально

2024–2025 годы — десятки тысяч новых CVE ежегодно

Просто запомнить всё невозможно → нужны системы учёта

# Как сообщают об уязвимостях

Стандарт — **Timed Coordinated Disclosure**

1. Исследователь находит уязвимость
2. Приватно сообщает вендору
3. Вендору даётся **embargo** — 14–90 дней на фикс
4. После срока или после патча — публичное раскрытие

Подготовьтесь заранее:
security@ваш-домен.ru + vulnerability policy на сайте

# CVE — Common Vulnerabilities and Exposures

Глобальный словарь публичных уязвимостей

Формат: **CVE-ГОД-НОМЕР**

Пример: **CVE-2014-0160** — Heartbleed в OpenSSL

- CVE выдают не только MITRE — десятки **CNA** (Microsoft, Red Hat, GitHub и др.)
- Не все публичные уязвимости получают CVE (нужно запросить + ПО должно быть публично выпущено)

# CWE — Common Weakness Enumeration

CWE — каталог **типов** уязвимостей (классов)

| CVE                     | CWE       | Что описывает                                |
|-------------------------|-----------|----------------------------------------------|
| CVE-2024-12345          | CWE-79    | Конкретная XSS в Joomla 5.0                  |
| CVE-2014-0160           | CWE-125   | Heartbleed (чтение за границы буфера)        |

CWE помогает понять, как предотвращать целый класс проблем

# Самые опасные типы уязвимостей

Два главных списка:

1. **OWASP Top 10** — критичные риски веб-приложений
2. **CWE Top 25** — самые распространённые слабости во всём ПО

Есть и специализированные:

- OWASP Mobile Top 10
- OWASP API Top 10
- OWASP IoT Top 10

Повышеное внимание к уязвимостям из этих списков

# Почему знание «топовых» уязвимостей так важно?

90–99 % реальных инцидентов — ошибки из OWASP Top 10 / CWE Top 25

Если просто перестать их допускать — количество уязвимостей упадёт в десятки раз

Изучать всё невозможно
Изучать самое частое и опасное — реально и эффективно

# Отсутствие «топовых» уязвимостей ≠ полностью безопасное ПО

Это необходимо, но недостаточно

Плюсы знания «топовых» ошибок:

- Сильно снижаете количество багов
- Тренируете «мышление атакующего»
- Легче замечать
